# Глава 2. Сортировка выбором

## Как работает память

Память компьютера состоит из множества ячеек, каждая из которых может хранить одно значение.  

Для хранения набора значений используются структуры данных, такие как **массивы** и **связанные списки**.

---

## Массивы и связанные списки

### Массив

**Массив** — структура данных, в которой элементы хранятся в памяти **непрерывно**, друг за другом.

#### Преимущества массива: 

**Произвольный доступ** — заранее известен адрес каждого элемента, поэтому по индексу можно мгновенно получить любой элемент массива `(O(1))`.

#### Недостатки массива: 
- **Сложность изменения размера**.  
При добавлении или удалении элементов иногда приходится сдвигать оставшиеся элементы в памяти.
-	**Выделение нового места**.  
Если следующая ячейка занята, нужно выделять новый участок памяти и копировать все элементы.  
Возможное решение — резервирование дополнительного места:
	- Плюс: не придeтся часто выделять память заново.
	- Минус: часть памяти может остаться неиспользованной, а при переполнении всё равно придeтся перемещать элементы.


Операция|Время выполнения
-|-
Чтение|O(1)
Вставка|O(n)
Удаление|O(n)

---

### Связанный список

**Связанный список** — структура данных, элементы которой могут храниться в памяти в любом месте.    
Каждый элемент содержит значение и ссылку на следующий элемент списка.

#### Преимущество связанного списка: 

Вставка и удаление элементов быстрее, чем в массивах.

#### Недостаток связанного списка: 

Отсутствие **произвольного доступа**: для получения элемента по индексу нужно пройти список с начала.

Операция|Время выполнения
-|-
Чтение|O(n)
Вставка|O(1)
Удаление|O(1)

---

## Сортировка выбором

**Сортировка выбором** — алгоритм сортировки массива, который на каждом шаге находит **минимальный (или максимальный)** элемент и помещает его в отсортированную часть массива.

### Принцип работы алгоритма:

1. Находит минимальный элемент в неотсортированной части массива  
2. Перемещает его в начало (или в новый массив)  
3. Повторяет процесс для оставшихся элементов  

С каждым шагом неотсортированная часть уменьшается, а отсортированная — увеличивается.

#### Например:

Исходный массив: `[5, 3, 4, 1, 2]`

Шаги:
- минимальный элемент — `1` → `[1]`
- затем `2` → `[1, 2]`
- затем `3` → `[1, 2, 3]`
- и так далее

Результат: `[1, 2, 3, 4, 5]`

**Время выполнения**: `O(n²)`.  
Сортировка выбором выполняет **поиск минимального элемента** за `O(n)` и повторяет этот процесс `n` раз.

Вернее было бы сказать, что время выполнения алгоритма составляет `O(n * 1/2 * n)`, но константы в O-большом **игнорируются**.

**Пример сортировки списка с помощью алгоритма сортировка выбором**  

[Код функции `selectionSort`](./selection-sort.js)

```
function findSmallest(arr) {
  let smallest = arr[0];
  let smallestIndex = 0;
  for (let i = 0; i < arr.length; i++) {
    if(arr[i] < smallest) {
      smallest = arr [i];
      smallestIndex = i;
    }
  }
  return smallestIndex;
}

function selectionSort(arr) {
  let newArr = [];
  let length = arr.length;
  for (let i = 0; i < length; i++) {
    let smallest = findSmallest(arr);
    newArr.push(arr.splice(smallest, 1)[0]);
  }
  return newArr;
}

const unsortedList = [5, 3, 4, 1, 2];
const sortedList = selectionSort(unsortedList);

console.log(sortedList); // [1, 2, 3, 4, 5]
```
